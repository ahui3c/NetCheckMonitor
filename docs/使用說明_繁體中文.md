# 對外網路連線能力監控程式 NetCheckMonitor 使用說明（繁體中文）

版本：0.9.6

NetCheckMonitor 可定時監控對外網路連線，記錄斷線並產生圖文報表，支援 Google Drive 備份與 PDF 下載。程式完全免費、開源、無廣告。

## 介面語言

- 程式第一次執行時會要求選擇「繁體中文」或「English」，選擇後會記住設定，日後不再重複詢問。
- 可在主畫面的「設定」中手動變更介面語言；新語言會在下次啟動程式時套用。
- 程式不再依照 Windows 系統語言自動判斷，也不需要下載語言檔。

## 開始監控

1. 執行 `NetCheckMonitor.exe`。
2. 預設每 60 秒檢查一次，可在開始前修改間隔。
3. 如需改用自己的目標，按主畫面下方「設定」，選擇「使用自訂測試目標」並輸入一至三組網站或 IP。
4. 按「開始監控」。最小化視窗後仍會在系統匣持續執行。
   首次失敗會在 5 秒後快速複查，連續失敗才會確認斷線；長時間斷線會自動降低複查頻率，但檢查間隔不會超過主畫面設定的週期。
   監控期間系統匣圖示會保持顯示：綠色代表連線正常、紅色代表確認斷線、橘色代表檢查中、灰色代表暫停。
   主畫面會顯示目前網卡、連線類型（有線／Wi-Fi／VPN）及 Wi-Fi 訊號百分比；偵測到網卡切換時會更新並寫入 CSV、HTML 與 PDF 報表。
5. 暫時不想納入測試時按「暫停」；恢復時按「繼續」。暫停區段會留下標示，但不計入統計。
6. 按「結束並產生報表」建立累積 HTML 報表，內容包含所有尚未清除的歷史監控資料。

## 事件註記

- 監控開始後可按主畫面下方的「事件註記」，暫停期間也能使用。
- 可自行輸入最多 500 個字元，或快速填入「重開數據機」、「重開無線路由」、「電腦重新開機」、「下雨」、「打雷」。
- 註記內容與當下時間會立即寫入主要 CSV 與本機備援，並整合到報表的「斷線事件與事件註記」表格；24 小時時間軸會以紫色標線呈現。
- 在 HTML 報表中將滑鼠靠近連線狀態或紫色註記標線，可查看對應時間、狀態與註記內容。
- 事件註記只作為時間標記，不會改變斷線次數、斷線時間或百分比統計。

## 監控目標設定

- 「使用內建測試目標」會依序測試 Microsoft、Google 與 Cloudflare 的公開連線端點。
- 「使用自訂測試目標」可設定最多三組網站或 IP，依畫面順序嘗試；任一成功即判定本次對外連線正常。
- 可輸入完整 `https://example.com/status`、一般網站 `example.com` 或 IP `1.1.1.1`。未填通訊協定時，網站自動使用 HTTPS，IP 自動使用 HTTP。
- 僅支援 HTTP 與 HTTPS，不接受內含帳號密碼的網址；空白、格式錯誤與重複目標會在儲存前提示。
- 設定保存於目前 Windows 使用者的 `%LOCALAPPDATA%\NetCheck\Monitor\settings.json`。
- 可分別勾選「登入 Windows 後自動啟動程式」及「程式啟動後自動開始監控」。若有未完成工作階段，程式會先詢問是否接續，再決定是否建立新的自動監控。
- 程式啟動時會檢查是否已有 NetCheckMonitor 執行中；重複啟動只會顯示既有視窗，不會建立第二份監控。
- 監控期間仍可開啟「設定」。若測試目標有變更，程式會先安全結束並保存原工作階段與報表，再以新目標自動開始新的監控；只修改自動啟動選項不會中斷目前監控。
- 可分別設定「監控期間防止電腦進入休眠」與「監控期間阻止 Windows 關機或重新啟動」。休眠保護預設開啟，關機保護預設關閉；啟用關機保護後請先停止監控或按「關閉程式」再關機。Windows 強制更新、斷電或硬體重置仍可能中斷程式。
- 可選擇「HTTPS 失敗時執行進階分層連線診斷」。正常成功時不增加測試；失敗時才檢查網卡、預設閘道、DNS、IPv4、IPv6、HTTPS 目標與 Wi-Fi 訊號。監控中切換會立即生效，不需重新開始。

## 即時報表與 PDF

- 監控期間可隨時按「產生即時報表」，檢查不會中斷。
- 程式每 10 分鐘自動更新 `_Cumulative_Live.html` 累積即時報表。
- 即時與結束 HTML 報表會彙整所有尚未清除的 CSV，而不是只統計本次執行。
- 程式未執行、電腦關機、不同工作階段之間的空白時間，以及完全沒有檢查紀錄的工作階段，都不計入有效監控時間與斷線百分比。
- 「下載報表 PDF 文件」可選擇全部資料或指定起訖日期。
- PDF 包含每日有效監控時間、每日估計斷線時間、斷線百分比、事件表與 24 小時時間軸。
- PDF 使用 Windows 內建 Microsoft Edge 的無介面列印功能。
- 報表另包含斷線事件數、最長／平均／最短斷線、第 95 百分位與最高延遲，以及平均延遲變動。

## 當機與重新開機後接續

- 監控中的工作狀態會寫入 `%LOCALAPPDATA%\NetCheck\Monitor\active-session.json`，每次檢查、暫停及繼續後立即更新。
- 程式異常關閉或 Windows 重新開機後，再次啟動會詢問是否接續原本 CSV。
- 接續前程式未執行的區段會標示為中斷，並從有效監控時間與斷線百分比排除。
- 使用「結束並產生報表」或安全關閉時，接續狀態會正常清除。

## Google Drive 每日備份

1. 按「Google Drive 備份設定」。
2. 按「登入 Google Drive」，在系統瀏覽器登入自己的帳號並同意權限。
3. 設定並儲存每日備份時間；預設為 23:55。
4. 第一次連接會在 Drive 建立或沿用 `Net_Check` 資料夾。
5. 每日上傳完整 PDF 與原始 CSV；同名檔案會更新，不會一直產生重複檔案。

程式只要求 `drive.file` 權限。登入權杖由 Windows DPAPI 加密保存在目前 Windows 帳號中。程式必須保持執行才能準時備份；錯過時會在下次啟動優先補傳最近一天。

## 輸出檔案與識別碼

- 主要紀錄位於程式旁的 `NetCheck_Data`。
- 無法寫入程式目錄時，改存到「文件\NetCheck_Data」。
- 復原副本位於 `%LOCALAPPDATA%\NetCheck\Recovery`。
- 檔名使用 `NetCheck_電腦名稱-8碼識別碼_日期時間`。
- 8 碼識別碼優先由 Windows MachineGuid 單向雜湊產生，讀不到時才使用 MAC 位址或電腦名稱雜湊。
- 原始 MachineGuid、MAC 位址與硬體序號不會寫入 CSV、報表或檔名。

## 長時間執行與資料保護

- 每筆結果立即強制寫入主要 CSV 與復原副本。
- 按視窗右上角 X 或 Alt+F4 只會縮到系統匣，不會結束程式。第一次操作時會提醒使用右下角「關閉程式」安全結束；提醒只顯示一次。
- 按右下角「關閉程式」時，會先停止監控、保存資料並建立最終報表。
- 監控時會要求 Windows 不要自動睡眠，但螢幕仍可依系統設定關閉。
- 筆記型電腦建議接上電源；闔上螢幕若設定為睡眠，Windows 仍可能進入睡眠。
- 請勿在監控期間直接拔除存放程式的 USB 隨身碟。

## 清除資料

主畫面右下角的「清除儲存資料」會刪除 NetCheckMonitor 管理的 CSV、HTML、即時報表及復原副本。只有執行此功能後，後續累積報表才會從空白資料重新開始統計。下載到其他資料夾的 PDF 不會被刪除。監控或雲端備份進行中時，清除功能會取消以保護資料。

## 斷線百分比計算

每日斷線百分比＝該日確認斷線時間 ÷ 該日有效監控時間 × 100%。暫停、程式中斷與關機時段會從兩者排除。首次失敗不會立即算成斷線，必須經快速複查確認。

進階診斷的開啟或關閉只影響失敗後是否收集分層證據，不影響斷線判定及所有統計，因此不同設定產生的連線率、斷線時間與百分比仍可直接比較。未啟用期間的失敗會在報表標示為「未執行進階診斷」。

## GitHub 與版本檢查

- 「關於」頁提供 GitHub 專案連結及「檢查新版本」按鈕。
- 只有使用者按下按鈕時才會向 GitHub Releases API 查詢一次，不會在背景定時連線。
- 發現新版時只會詢問是否開啟 Releases 頁面，由使用者自行決定是否下載；程式不會自動下載或安裝。

## 系統需求

- Windows 10 或 Windows 11
- .NET Framework 4.8
- Microsoft Edge（PDF 產生）
- Google Drive 備份需要網路及可用的預設系統瀏覽器
